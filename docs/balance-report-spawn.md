# 콜렉션 모드 밸런스 리포트 — 스폰 전략과 난이도

> 질문: **"블록이 무조건 size 1부터 나오는 게 더 어려운가, 아니면 보드 최고값에 가까운
> 큰 수가 자주 나올수록 난이도가 어떻게 변하는가?"**
>
> 결론 먼저: **"항상 size 1 스폰"이 압도적으로 어렵다.** 큰 수가 자주 나올수록 쉬워진다.
> (직관과 반대로 느껴질 수 있으나, 시뮬레이션·게임 이론 모두 일관된 결과.)

---

## 1. 시뮬레이션 설정

- 보드: **4×4, sizeOnly 머지** (색 무관, 같은 size끼리 합쳐져 2배) — 실제 콜렉션 모드와 동일 규칙.
- 목표: size **1024** 도달 = 테마 완성.
- 두 스폰 전략 비교:
  - **always1**: 머지 후 새 블록은 **항상 size 1** (현재 구현).
  - **near-max**: 보드 최고값에 가까운 수가 더 자주 스폰 (가중 랜덤: `1`(w1) / `max/4`(w2) / `max/2`(w3)).
- 3종 AI로 교차 검증 (편향 방지):
  - **greedy**: 매 턴 머지 수 최대화(+빈칸 보조) — 숙련 플레이어 상한.
  - **corner**: down→left 우선의 코너 몰기 — 일반적 사람 플레이.
  - **random**: 가능한 방향 무작위 — 하한/무전략.
- 표본: AI×전략당 500판, seed 고정(재현 가능).

---

## 2. 결과

### AI별 비교 (500판)

| AI | 전략 | 평균 턴 | 평균 도달단계(log2) | 중앙값 최고 size | 1024 완성률 |
|----|------|--------:|--------------------:|-----------------:|------------:|
| greedy | **always1** | 273.2 | 6.64 | **128** | **0.0%** |
| greedy | near-max    | 49.1  | 8.04 | 256 | 21.8% |
| corner | **always1** | 236.2 | 6.40 | **64**  | **0.0%** |
| corner | near-max    | 47.0  | 6.83 | 128 | 4.0% |
| random | **always1** | 134.7 | 5.63 | **64**  | **0.0%** |
| random | near-max    | 47.6  | 6.59 | 64  | 3.6% |

> 평균 도달단계는 `log2(최고 size)` 평균. 예: 6.64 ≈ size 100 부근, 8.04 ≈ size 260 부근.

### 핵심 수치

- **always1은 어느 AI로도 1024 완성률 0%.** 숙련(greedy)조차 중앙값 128에서 막힘.
- **near-max는 greedy 기준 완성률 21.8%**, 사람 플레이(corner)도 4% 도달.
- always1은 판당 턴이 5~6배 길다(273 vs 49). → **길게 끌지만 끝내 못 깬다.**

## 3. 왜 "size 1만 스폰"이 더 어려운가

핵심은 **보드 칸 수(16칸)는 유한한데, 1024를 만들려면 size 1짜리 1024개를 합쳐야** 한다는 점.

1. **머지 효율 격차.**
   1→1024는 **10단계 머지**가 필요(1→2→4→…→1024). 한 번 합칠 때마다 칸 하나가 비지만,
   매 머지 후 새 size 1이 또 들어와 **저단계 블록이 계속 쌓인다.**
   near-max는 큰 수가 직접 들어와 **상위 단계를 건너뛰며 빠르게 합쳐진다.**

2. **칸 고갈(deadlock).**
   16칸 보드에서 1024 하나를 만드는 동안 1~512 부산물이 칸을 점유.
   size 1이 계속 보충되면 **정리 속도 < 유입 속도** → 보드가 잠겨 게임오버.
   (always1의 긴 평균 턴 273 = "오래 버티지만 결국 막힘"의 증거.)

3. **저단계 블록의 함정.**
   size 1·2가 흩뿌려지면 큰 블록끼리 인접시키기 어려워 머지 체인이 끊긴다.

> **요약:** always1 = "엔진 출력은 약한데 매번 짐(저단계)을 더 싣는" 구조.
> near-max = "큰 부품을 직접 공급해 조립을 가속"하는 구조.

## 4. 게임 디자인 관점 권고

현재 콜렉션은 **테마 완성(1024 도달)이 진행/해금의 핵심**이다.
always1로는 일반 유저의 완성률이 사실상 0% → **진행 불가 → 이탈 위험.**

### 권고: 진행도 연동 가중 스폰 (난이도 곡선 완만화)

보드 최고값에 비례해 **점진적으로 더 큰 시작 블록**을 섞는다. 예:

| 보드 최고 size | 스폰 분포(예시) |
|----------------|-----------------|
| ≤ 16  | size 1 100% (초반 학습) |
| 32–128 | size 1 70% / 2 20% / 4 10% |
| ≥ 256 | size 1 50% / 4 30% / 8 20% |

- 초반엔 "1부터 키우는" 명확한 학습 경험 유지.
- 후반엔 칸 고갈을 완화해 **완성이 현실적**이 되도록.
- "최고값에 가까운 수"를 직접 주기보다 **한두 단계 아래**를 섞는 게 밸런스가 부드럽다
  (near-max의 `max/2` 직공급은 21.8%로 다소 쉬워짐 — 보상형 CONTINUE와 합쳐지면 너무 쉬울 수 있음).

### CONTINUE(광고) 기능과의 상호작용

- 현재 CONTINUE = "상위 3종 외 제거"는 **칸 고갈 해소** 그 자체 → always1의 막힘 문제를 부분적으로 구제.
- 즉 always1 유지 + CONTINUE 적극 노출 = "광고로 진행을 사는" 수익 모델로도 성립.
  단 **광고 없이도 깰 수 있는 최소 경로**는 남겨야 이탈을 막는다 → 가중 스폰 병행 권장.

## 5. 결정 필요 사항

| 옵션 | 효과 |
|------|------|
| A. always1 유지 | 가장 어려움. 완성은 사실상 CONTINUE(광고) 의존. 수익↑·이탈 위험↑ |
| B. 진행도 연동 가중 스폰(권장) | 완만한 난이도 곡선. 무과금 완성 경로 확보 + 후반 가속 |
| C. near-max 풀적용 | 가장 쉬움(완성률↑). 도전감↓, CONTINUE 가치↓ |
