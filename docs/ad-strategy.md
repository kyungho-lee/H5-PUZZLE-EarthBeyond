# NeonDrift — 광고 수익 전략 (콜렉션 모드)

> 목표: **점유율(리텐션) 유지**와 **리워드 광고 수익**을 동시에 만족.
> 원칙: 강제 노출은 최소화하고, 보상이 명확한 지점에서 유저가 **스스로 광고를 선택**하게 한다.

## 광고 종류 매핑

| 위치 | 광고 종류 | 트리거 | 보상/효과 | 실패 시 |
|------|-----------|--------|-----------|---------|
| 게임오버 진입 | **전체광고(interstitial)** | 자동 (팝업 직전 1회) | 없음 (노출형) | 그냥 팝업 표시 |
| 게임오버 1단계 | **스타 소모(부족 시 리워드)** | `CONTINUE (⭐N 또는 📺AD)` | 상위 3종 외 블록을 터지는 연출 후 제거하고 이어하기. 비용 5→10→20→40(회수당). **잔액 부족 시에만 광고 대체** | 광고 거부 시 이어하기 차단 |
| 게임오버 2단계 | **리워드** | `📺 DOUBLE STARS` 버튼 | 이번 판 획득 스타 **x2** (추가 1배 적립) | 보상 없음, 버튼 복구(재시도 가능) |
| 테마 완성 화면 | **리워드** | `📺 DOUBLE REWARD` 버튼 | 완성 보상 **⭐20 → ⭐40** | 보상 없음, 버튼 복구 |
| RETRY (FREE) | **없음** | `RETRY (FREE)` 버튼 | 무료 재시작 | — |

### 게임오버 화면 2단계 흐름 (v0.5.1)

광고 버튼을 한 번에 하나만 노출해 의미 충돌·광고 피로를 줄인다.

1. **1단계 — CONTINUE 권유**: `📺 CONTINUE` + `END RUN ▸`. (제거할 블록이 있을 때만 진입)
2. **2단계 — 정산**: `END RUN ▸` 클릭 또는 CONTINUE 불가 시 → `THIS RUN ⭐` + `📺 DOUBLE STARS` + `RETRY (FREE)`.
- GALLERY/MENU는 두 단계에 공통 노출. `collectionSetOverPhase()`로 전환.

### CONTINUE (리워드형) 상세

- **유지 기준**: 보드에 실제 존재하는 size 중 내림차순 **상위 3종**만 남기고 나머지 셀 제거.
  (예: 보드에 `256,128,64,32,16,8`이 있으면 256/128/64 유지, 32/16/8 제거)
- **연출 순서**: 광고 시청 → 제거 대상 셀에 **터지는 연출(파티클·링·플래시)을 먼저** 보여줌
  → 약 0.65초 후 실제 제거 → 보드 갱신 → 계속. 효과를 확실히 인지시키는 게 목적.
- **무료(스타 0)**: 광고만 시청하면 됨. 제거할 블록이 하나도 없으면(상위 3종 이하) 버튼 숨김.
- 구현: `continueRemovableCells()` / `collectionContinueWithAd()` / `playClearFx()` (index.html).

## 핵심 정책

1. **RETRY는 항상 무료.** 광고로 막지 않는다 → 이탈 방지, 세션 길이↑.
   대신 게임오버 *진입 시* 전체광고를 1회 노출해 노출형 수익을 확보한다.
2. **x2 스타는 100% 리워드형(자발적).** 기본 1배는 즉시 적립되고,
   광고를 보면 같은 양을 한 번 더 적립해 총 x2. **광고 실패/거부 시 1배는 유지**된다.
   → 유저는 손해 없이 "추가 이득"만 선택하므로 거부감이 낮고 시청률↑.
3. **빈도 제한 없음 (명시적 액션만).** 모든 리워드 광고는 버튼 클릭으로만 호출되므로
   스팸이 아니다. 인위적 쿨다운을 두지 않아 적극적 유저의 수익 기여를 극대화.
4. **게임오버 1회당 x2는 1번만.** `_gameoverStarsDoubled` 플래그로 중복 차단.
   테마 완성도 `_themeStarsDoubled`로 1회 제한.

## 스타 이코노미 정합성

- 콜렉션 모드 스타 적립은 일일 캡 면제(`earnStarsToWallet(..., 'collection', ...)`).
  → x2 추가분이 캡에 막히지 않는다.
- CONTINUE 비용은 진행도에 비례(`min(5 + step*2, 30)`). x2로 번 스타로 CONTINUE 가능.
- 갤러리 NEW 블록 수령 시 ⭐1(블록당 1회 영구)은 별도 소액 리워드 — 광고 아님.

## Playgama placement ID

`src/playgama-bridge-config.json` 의 `advertisement.rewarded.placements`:

| ID | 용도 |
|----|------|
| `daily_retry` | (기존) 데일리 재시도 |
| `retry_reward` | (기존) 재시도 보상 |
| `collection_double_stars` | 게임오버 스타 x2 |
| `collection_theme_double` | 테마 완성 보상 x2 |
| `collection_continue` | 게임오버 이어하기(상위 3종 외 제거) |

> 새 리워드 지점을 추가할 때는 **이 표 + config의 placements + 코드의 `requestRewardedAd('<id>')`** 세 곳을 함께 갱신.

## 향후 확장 여지 (미구현)

- **CONTINUE 무료화(광고형)**: 스타 부족 시 "광고 보고 1회 이어하기" 옵션. 단 난이도 밸런스 검토 필요.
- **데일리 보너스 스타**: 하루 1회 광고로 ⭐ 지급(리텐션 훅).
- **애니메이션 이미지팩 언락**: 프리미엄 스킨을 리워드 광고 N회 또는 결제로.
