# 콜렉션 테마 블록 이미지팩 — 에셋 제작 가이드

> NeonDrift 콜렉션 모드의 "테마"는 단순한 갤러리 리워드가 아니라
> **보드에서 머지하며 성장하는 10단계 블록 스킨(이미지팩)** 이다.
> 블록 타일 자체가 테마 이미지로 꽉 차게 렌더링되며, 마지막 단계(1024 머지)
> 도달 시 테마가 완성되고 다음 테마(다른 블록 스킨)가 해금된다.

---

## 1. 개념

- **테마 = 블록 스킨 1세트** (10단계 이미지 + 완성 트리거)
- 머지로 size가 커질수록 블록 그림이 다음 단계로 "성장"한다.
- 예) **행성 테마**: 수성 → 금성 → 지구 → 화성 → 목성 → 토성 → 천왕성 → 해왕성 → 명왕성 → 태양 (→ 은하계=완성)
- 향후 **애니메이션 이미지팩**(움직이는 블록)을 추가 리워드로 확장 가능.

### 단계 ↔ 머지 크기 매핑 (현재 코드 기준)

| step | 머지 size | 비고 |
|-----:|----------:|------|
| 01 | 1 | 시작 블록 (스폰 단위) |
| 02 | 2 | |
| 03 | 4 | |
| 04 | 8 | |
| 05 | 16 | |
| 06 | 32 | |
| 07 | 64 | |
| 08 | 128 | |
| 09 | 256 | |
| 10 | 512 | |
| 11 | **1024** | **최종 단계 = 테마 완성 트리거** (갤러리에 포함) |

> v0.7부터 1024도 갤러리 단계(step-11)다. 단계 수는 `collection.js`의 `STEP_SIZES`가 단일 소스이며,
> 추가하려면 그 배열에 size를 넣고 각 테마에 `step-NN.svg`만 추가하면 갤러리·완성판정·HUD가 자동 반영된다.

> 매핑 소스: `src/collection.js` 의 `SIZE_TO_STEP` / `STEP_SIZES`.
> 코드를 바꾸지 않는 한 위 표를 기준으로 에셋을 제작한다.

## 2. 파일 규격

### 디렉터리 / 네이밍

```
src/themes/<theme-id>/
  step-01.<ext>   # size 1
  step-02.<ext>   # size 2
  ...
  step-10.<ext>   # size 512
```

- `<theme-id>`: 케밥케이스 (`neon-digits`, `planets`, `cyber-ocean` …)
- 파일명은 **반드시 `step-01` ~ `step-10`** (2자리 zero-pad). 코드가 인덱스로 직접 참조한다.
- 경로 등록은 `src/collection-themes.js` 의 `makePaths(id)` 가 자동 생성하므로,
  새 테마는 `CollectionThemes` 배열에 항목만 추가하면 된다.

### 권장 포맷: **WebP** (최적화 우선)

| 항목 | 값 |
|------|-----|
| 포맷 | `.webp` (정적), 애니메이션은 animated WebP |
| 캔버스(정사각) | **256 × 256 px** (권장). 최소 192, 최대 512 |
| 종횡비 | 1:1 고정 (블록 셀이 정사각) |
| 배경 | **투명** 권장. 불투명이면 블록 셀과 같은 둥근 사각 영역 안에 그림이 꽉 차게 |
| 색공간 | sRGB |
| 파일 크기 | 정적 1장당 **≤ 20 KB** 목표 (≤ 40 KB 상한) |
| 애니메이션 | ≤ 60 KB / 장, 12~24 fps, 1~2초 루프 |

> **왜 256px인가?** 보드 셀은 디바이스에 따라 ~80–125 CSS px,
> 레티나(@2x/@3x)에서 최대 ~375 물리 px까지 확대된다.
> 256px이면 일반 기기에서 선명하고, 512px은 고해상 태블릿 대응 여유분.
> 용량(H5 로딩 속도)과 선명도의 균형점으로 256px을 기본 권장한다.

### 대체 포맷

- **SVG**: placeholder/벡터 아이콘에 적합 (무한 확대, 초경량). 단 복잡한 일러스트·그라데이션은 용량이 커지고 렌더 비용↑. → **초기 placeholder는 SVG, 정식 아트는 WebP**.
- **PNG**: WebP 미지원 환경 fallback이 필요할 때만. 동일 화질에서 WebP 대비 1.5~2배 용량.

## 3. 디자인 가이드

### 블록 한 칸을 꽉 채운다
- 셀 안쪽 여백은 렌더러가 `cell * 0.86` 으로 이미 처리한다.
  → **이미지 자체는 캔버스 가장자리까지 꽉 채워** 제작 (이미지 내부 여백 최소화).
- 둥근 모서리는 렌더러가 자르지 않으므로, 모서리까지 보이길 원하면 투명 배경 + 둥근 형태로 디자인.

### 단계별 "성장" 가독성
- 인접 단계(step N ↔ N+1)는 **한눈에 구분**되어야 한다 (색/형태/크기감 변화).
- 행성처럼 "점점 커지는" 모티프면 단계가 올라갈수록 화면 점유율·디테일·발광을 키운다.
- 최종 단계(step-10)와 완성(1024)은 가장 화려하게 — 클리어 보상감.

### 네온 톤 일관성
- 게임 전체가 사이버펑크 네온 톤. 테마별 대표 발광색 1~2개를 정해 통일.
- 배경이 어두우므로(거의 검정) **밝은 발광·외곽선**이 잘 살아난다.

## 4. 새 테마 추가 절차

1. `src/themes/<theme-id>/` 폴더 생성, `step-01`~`step-10` 에셋 배치.
2. `src/collection-themes.js` 의 `CollectionThemes` 배열에 항목 추가:
   ```js
   {
     id: 'planets',
     label: 'PLANETS',
     description: '머지로 자라나는 태양계',
     unlockCondition: 'neon-digits',   // 이전 테마 완성 시 해금 (첫 테마는 null)
     svgPaths: makePaths('planets'),    // step-01~10 경로 자동 생성
   }
   ```
   > `makePaths`는 현재 `.svg` 확장자를 생성한다. WebP로 전환 시
   > `makePaths`의 확장자 또는 테마별 `ext` 옵션을 함께 갱신할 것.
3. 첫 테마는 `unlockCondition: null`, 나머지는 직전 테마 id를 지정해 체인 구성.
4. 렌더러(`grid-render.js`)는 `collTileImage(theme, size)` 로 size→step→경로를
   자동 로드하므로 **추가 렌더 코드 불필요**.

## 5. 행성 테마 (planets) — 제작 사양 예시

이미지 생성이 가능해지면 아래 사양으로 제작한다. 그 전까지는 미제작.

| step | size | 행성 | 대표 발광색 | 모티프 |
|-----:|-----:|------|------------|--------|
| 01 | 1 | 수성 (Mercury) | `#9a8c7a` | 작고 회갈색, 크레이터 |
| 02 | 2 | 금성 (Venus) | `#e6b85c` | 노란 구름층 |
| 03 | 4 | 지구 (Earth) | `#3fa9f5` | 파란 바다 + 초록 대륙 + 구름 |
| 04 | 8 | 화성 (Mars) | `#e2683c` | 붉은 사막, 극관 |
| 05 | 16 | 목성 (Jupiter) | `#d8a25a` | 줄무늬 + 대적점 |
| 06 | 32 | 토성 (Saturn) | `#e8d49a` | 고리 (셀 안에 들어오게) |
| 07 | 64 | 천왕성 (Uranus) | `#7fd4d4` | 청록, 옆으로 누운 고리 |
| 08 | 128 | 해왕성 (Neptune) | `#3c5fe2` | 진한 파랑, 폭풍 |
| 09 | 256 | 명왕성 (Pluto) | `#c9b8a0` | 하트 무늬(톰보 영역) |
| 10 | 512 | 태양 (Sun) | `#ff8a00` | 강한 코로나 발광 |
| — | 1024 | 은하계 (Galaxy) | `#b06aff` | 완성 연출용 나선 은하 |

> 1024(은하계)는 보드 블록이 아니라 **완성 팝업 연출**에 쓰는 별도 이미지.
> 필요 시 `step-10` 과 동일 폴더에 `complete.<ext>` 로 추가하고 완성 팝업에서 참조.

## 6. 현재 placeholder 상태 (neon-digits)

- `neon-digits` 는 **숫자만** 표기한 placeholder 테마.
- 각 step SVG는 해당 size 숫자(1,2,4,…,512)를 네온 글로우로 큼직하게 렌더.
- 보드 블록도 이 SVG로 그려지며, 렌더러는 이미지 위에 숫자를 **중복 표기하지 않는다**
  (이미지에 숫자가 이미 포함). → `grid-render.js` `drawTile` 의 `drewImage` 분기.
- 정식 그림 테마(행성 등)가 준비되기 전까지 기능 테스트용 기준 테마로 사용.

### placeholder 규격 (정식 에셋도 이 캔버스 기준 권장)
- viewBox `0 0 240 240` (정사각), 숫자/그림은 캔버스 거의 꽉 차게.
- step-10 은 "STEP 10 / 10" 등 최종 단계 표시, 완성감 있는 발광.
