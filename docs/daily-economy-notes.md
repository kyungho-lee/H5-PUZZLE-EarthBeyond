# Daily 모드 — 역할·리트라이·경제 논의 (v0.8.2)

> 결론: **Daily = 재화(별) 벌이 채널, 콜렉션 보조.** 리트라이는 "더 잘해서 더 벌기" 수단이며
> best-run 증분 적립으로 무한 수급을 막는다. 리트라이 5→3회로 축소.

## 1. 현재 구조 (사실)

| 항목 | 값 |
|------|-----|
| 보드 | 4×4, sizeOnly, 요일별 난이도(WARMUP…GAUNTLET 7종) |
| 별 보상 | 타깃 size 도달: 64=1⭐, 128=2⭐, 256=3⭐, 512=5⭐ (재획득 가능) |
| 리트라이 | **3회/일** (총 4판), RETRY는 리워드 광고(`daily_retry`) |
| 지갑 적립 | **오늘 best-run 증분만** `max(0, thisRun − prevBest)` (v0.8.1) |
| daily 총합 | `dailyState.stars` — 매 시도 누적(리더보드/표시용, 지갑과 분리) |
| 일일 지갑 캡 | `WALLET_DAILY_CAP = 80` |
| 리셋 | UTC 자정 (날짜 키 전환으로 자동) |

## 2. 역할 정의 — 재화 벌이 채널

- Daily의 산출물 = **별**. 사용처 = **콜렉션 CONTINUE**(스타 소모, 회수당 5→10→20→40)
  및 향후 재화 사용처.
- 따라서 Daily는 "오늘 한 판 잘 풀어 별을 번다" → 콜렉션 진행을 돕는 **공급 채널**.
- 리더보드 경쟁은 부수적(현재 Playgama 토큰 미연결로 DEMO).

## 3. 리트라이 — 왜 3회인가

- best-run 증분 적립이라 "리트라이 반복 = 무한 수급"은 **이미 차단**됨(반복해도 best를 못 넘으면 +0).
- 그럼에도 5회는: (a) 광고를 5번 권하는 피로, (b) 한 판의 긴장감 희석.
- → **3회**(총 4판)로 축소: 한 판의 무게↑, 광고 노출은 적정 유지, best 갱신 동기는 그대로.
- RETRY = 리워드 광고: "더 높은 점수에 도전 = 광고 시청"의 자발적 거래. 광고 거부 시 그날 종료.

## 4. 균형 포인트 / 향후 검토

- ~~**광고 보고도 +0 가능**~~ → **v0.9 해결**: 정산을 세션 종료 CLAIM 1회(best 전액)로 바꿔
  "광고로 best 도전 → CLAIM에서 그만큼 수령" 흐름 명확화. 광고 가치가 best 갱신으로 보장.
- **별 사용처 확장** → **v0.9 일부 반영**: 테마 즉시 해금(⭐50) 추가. 추가 사용처(부스터/스킨)는 향후.
- **일일 캡 80**: best-run 모델에선 사실상 그날 best(최대 ~수십)로 제한되므로 캡이 거의 안 걸림.
  사용처 확장 시 캡 재검토.
- **요일 난이도와 보상 연동**: 어려운 요일(GAUNTLET 등)에 보상 가중을 줄지 검토 여지.
