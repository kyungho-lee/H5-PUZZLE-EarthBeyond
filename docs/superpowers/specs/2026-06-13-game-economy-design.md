# NeonDrift — 게임 경제 GDD

**버전:** v0.2 (교차 리뷰 반영)
**작성일:** 2026-06-13
**기준 브랜치:** `feat/neon-drift-game`

---

## 0. 설계 원칙 (교차 리뷰 확정)

1. **단일 재화 노출** — 플레이어에게는 "별(⭐)" 하나만 보인다. 내부 분리는 숨긴다.
2. **광고 우선, 별 지불은 폴백** — 광고 로드 실패·플랫폼 미지원 시만 별 지불 활성화.
3. **리워드 광고 타이밍은 몰입 중** — 세션 시작 전 제안 금지. Collapse 직후 또는 게임오버 화면에서 제안.
4. **컨텐츠 소비 속도 제어** — 중급자도 테마 1개 해금에 5~7일 소요되도록 설계.
5. **광고 거부 플레이어도 진행 가능** — 느리지만 막히지 않는다.

---

## 1. 재화 구조

### 별(⭐) 2-tier — 플레이어에게는 단일 노출

```
플레이 중 별 획득 (기존 그대로)
       │
       ▼
[내부] dailyRunStars  ←  성과 기록 (일일 리더보드용, 기존 유지)
       │
       │ 세션 종료 시 이전 (earnStarsToWallet)
       ▼
[내부] neondrift_wallet.stars  ←  소비 가능한 누적 잔고
       │
       └─ UI 표시: 메뉴·Collection 화면에서만 "⭐ 47" 단순 표시
```

`dailyState.stars`는 건드리지 않는다. HUD의 "BEST" 슬롯은 기존 그대로 유지.

### localStorage 키

| 키 | 내용 |
|---|---|
| `neondrift_daily_YYYY-MM-DD` | 기존 유지 |
| `neondrift_wallet` | 신규 — `{stars, totalEarned, totalSpent, dailyEarnedToday, practiceEarnedToday, lastUpdated}` |
| `neondrift_buff` | 신규 — `{doubleStarSession: null}` (세션 ID 기반) |

## 2. 별 획득

### 기본 타겟 테이블 (코드 그대로 유지)

| 타일 | 기본 별 | 2배 버프 시 |
|---|---|---|
| 64 | ⭐×1 | ⭐×2 |
| 128 | ⭐×2 | ⭐×4 |
| 256 | ⭐×3 | ⭐×6 |
| 512 | ⭐×5 | ⭐×10 |

`remove:false` 유지 — 같은 타일 반복 생성 시 매번 지급.

### 실측 기반 기대 획득량 (daily-balance-sim.js 데이터 기준)

| 수준 | 한 판 기대 별 | 근거 |
|---|---|---|
| 입문 | ⭐6~8 | 64·128 위주 |
| 중급 | ⭐15~22 | 256 도달 |
| 숙련 | ⭐28~40 | 512 반복 |

> GDD v0.1의 "중급 ⭐28/판" 수치는 과대 추정. 실측 기반으로 수정.

### Daily 모드 Wallet 적립 상한

| 항목 | 한도 |
|---|---|
| 하루 Daily 별 적립 상한 | ⭐80 |
| 하루 Practice 별 적립 상한 | ⭐12 |
| 하루 총 상한 | ⭐92 |

> v0.1의 ⭐215 상한은 실질 의미 없음. 실측 중급자 하루 최대 ≈ ⭐60~80 기준으로 ⭐80으로 조정.

### Practice 모드 별 획득

- 허용하되 하루 ⭐12 상한 (Daily 1판 기대치의 약 60%)
- 상한 도달 후 조용한 캡 (애니메이션·적립 없음, 게임 플레이는 정상)

### Collection 모드 Collapse 보너스

| 이벤트 | 지급 | 조건 |
|---|---|---|
| Collapse 1회 (체인 1~9) | ⭐5 | 인터스티셜 광고 시청 완료 후 |
| 테마 완료 (체인 10) | ⭐20 | 완료 보너스 |
| 광고 거부 시 | ⭐2 | 완전 차단 금지, 축소 지급 |

### 리워드 광고 2배 버프

| 항목 | 값 |
|---|---|
| 지속 | 다음 1판 (세션 단위) |
| 적용 범위 | 해당 판의 타겟 별 획득 전체 |
| 중첩 | 불가 (기존 버프 → 다음 판으로 갱신) |
| **제안 타이밍** | **Collapse 직후** 또는 **게임오버 화면** |
| 세션 시작 전 제안 | **금지** — 수락률 최저 타이밍 |

버프 UI 문구 예시:
```
[Collapse 직후]
대박 연속! 다음 라운드 별 2배 받을게요?
[ 📺 광고 보기 ]
```

## 3. 별 소비

### Collection 테마 해금 비용

| 테마 순번 | 해금 비용 | 누적 | 중급자 누적 일수 기준 |
|---|---|---|---|
| 0 (기본) | 무료 | — | — |
| 1 | ⭐50 | ⭐50 | 약 3~5일 |
| 2 | ⭐90 | ⭐140 | 추가 5~7일 |
| 3 | ⭐130 | ⭐270 | 추가 8~10일 |
| 4 | ⭐180 | ⭐450 | 추가 10~14일 |

> v0.1의 ⭐30~100 비용은 중급자 2일 내 전 테마 소진 위험. 5~7일 기준으로 상향.
> Collection Collapse 보너스(테마당 ⭐50+⭐20=⭐70)가 다음 테마 비용의 일부를 충당하는 구조.

### Daily 재시도 비용

**원칙: 광고 우선, 별 지불은 폴백**

```
[재시도 팝업]

  ┌──────────────────────────────┐
  │   📺 광고 시청 후 재시도     │  ← 기본 CTA (크고 밝게)
  └──────────────────────────────┘

  또는 ⭐10 소모                   ← 소형 회색 텍스트 (광고 불가 시만 활성화)

  ✕ 포기
```

| 조건 | 별 지불 버튼 상태 |
|---|---|
| 광고 로드 가능 | **숨김** (광고 버튼만 표시) |
| 광고 로드 실패 | 활성화 — ⭐10 고정 |
| 플랫폼 광고 미지원 | 활성화 — ⭐10 고정 |
| 잔고 부족 | 비활성화 + "잔고 부족 ⭐N/⭐10" 표시 |

> v0.1의 ⭐8→22 누진 비용 폐기. 광고 폴백 고정 ⭐10으로 단순화.
> 이유: 누진 비용은 UI 복잡도 증가 + 이중 비용(광고+별) 구조로 재시도율 급감 위험.

## 5. 광고 전략

### 트리거 포인트

| 트리거 | 종류 | 별 연계 |
|---|---|---|
| Practice 게임오버 | 인터스티셜 | 없음 |
| Collection Collapse | 인터스티셜 | Collapse 보너스 ⭐5 (광고 완료 후) |
| Daily 재시도 | 리워드 | 재시도 허가 |
| **2배 버프** | **리워드** | **Collapse 직후 또는 게임오버 화면에서만 제안** |

### 광고 빈도 제한

- `lastInterstitialAt` 변수로 직접 관리 — 플랫폼 SDK 의존 금지
- Collapse 인터스티셜 최소 간격: **60초**
- 한 세션에 Collapse 3회 이상 → 2회차부터 인터스티셜 생략 (보너스 ⭐2만 지급)

### 광고 거부 플레이어 보장

| 상황 | 보장 경로 |
|---|---|
| Daily 재시도 광고 거부 | 별 지불 폴백 (⭐10) |
| Collapse 광고 거부 | 보너스 ⭐2 축소 지급 |
| 2배 버프 광고 거부 | 기본 배율 그대로 진행 |
| 리워드 전체 거부 | Daily 무제한+Practice로 별 자연 누적 |

## 6. 어뷰징 방지

### 실용적 방어 (H5 한계 인정)

| 방어 수단 | 구현 |
|---|---|
| 하루 상한 | `dailyEarnedToday ≤ 80`, `practiceEarnedToday ≤ 12` |
| 무결성 체크 | `stars ≤ totalEarned - totalSpent` — 불일치 시 Wallet ⭐0 리셋 |
| Firebase 이상치 플래그 | `submitScore` 시 `walletSnapshot` 포함, 서버 로그 기록 |

완전한 조작 방지는 H5 구조상 불가. 극소수 조작 허용 비용 < 방어 시스템 복잡도 비용.

## 7. 구현 우선순위

### Phase 1 — Collection 모드와 동시 구현

- `neondrift_wallet` localStorage 구조
- `earnStarsToWallet(runStars, mode)` 함수 (daily.js 확장)
- 메뉴·Collection 화면 "⭐ 47" 잔고 표시

### Phase 3 — 데이터 기반 튜닝

- 실제 Collapse 빈도·별 획득 분포 측정
- 일일 상한·테마 비용 재조정

## 9. 교차 리뷰 반영 기록

**밸런스 리뷰 지적 (v0.1→v0.2 주요 변경):**
- ⭐28/판 중급 과대 추정 → 실측 기반 ⭐15~22로 수정
- 테마 해금 비용 ⭐30~100 → ⭐50~180으로 상향 (2일 소진 방지)
- 하루 상한 ⭐215 → ⭐92로 하향 (실측 기반)
- ⭐8~22 누진 재시도 비용 → ⭐10 고정 폴백으로 단순화
- 2배 버프 "다음 1판 × 2배"는 ⭐200 cap이 있어도 왜곡 우려 → 제안 타이밍 이동으로 완화
- Collection 전용 재화 분리 → 단일 Wallet으로 유지, 인플레이션은 상한으로 제어

**UX/광고 수익 리뷰 지적 (v0.1→v0.2 주요 변경):**
- 2-tier 재화 플레이어 노출 → 단일 별 개념 노출, 내부 분리만 유지
- 세션 시작 전 버프 제안 → Collapse 직후·게임오버 화면으로 이동
- 별 지불 = 광고 대안 → 별 지불 = 광고 폴백으로 격하 (광고 가능 시 별 버튼 숨김)
- Collapse 광고 빈도 → 60초 쿨다운 직접 관리, 3회+ 생략 정책 추가
- 재시도 시각적 위계 명시 → 광고 CTA 크게, 별 지불 소형 텍스트
